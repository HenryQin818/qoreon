使用方式

先看什么
先看左侧问题怎样发生，再看右侧方案怎样改变阅读顺序。
这页不讲接口、不讲实现细节，只回答一个问题：用户为什么现在读不懂，改后为什么能第一眼看懂。
场景切换
点击任一步，就会同时刷新左右两侧的消息层级和数据指标。
拍板标准
  • 首屏先看到业务主语，而不是技术收纳件。
  • 当前结论 / 当前状态 / 下一动作 在首层稳定共存。
  • 反馈、过程、调试继续保留，但统一降到第二层。
一句话结论
这轮不是给聊天区加更多结构，而是把“谁在推进什么、现在什么状态、下一步谁接”重新拉回首屏第一层。
现状并非没展示，而是展示顺序偏向摘要、回执和技术收纳；方案态要把业务主语重新钉在用户第一眼能看到的位置。
本轮只做拍板
不改 live 聊天区 不新增接口 不改运行时装配 只看表达顺序
变更边界
本轮只动表达顺序，不扩成新消息体系
不增加新 viewer，不加新卡片层，不重做 chat layout。方案只回答“首屏先给什么”，不回答“再新造什么”。
仍然保留
反馈、过程、调试都继续存在
它们不是删除，而是退到第二层。用户要下钻时还能看到，只是不该继续抢走首屏主语和下一动作。
拍板点
只需要确认这条表达规则是否采纳
点击任一步，确认右侧是否比左侧更容易判断“谁接下一棒”。如果答案是肯定的，这页就足够支撑进入真实修复批。
