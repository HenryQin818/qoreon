# 公开公共技能包清单

这份清单说明 `standard_project` 只内置 8 个公开安全的公共 skill。

角色相关的信息不做 role skill，而是放在：

- 对应通道目录
- `seed/ccr_roster_seed.json`
- `主体-总控/产出物/沉淀/03-标准项目通讯录与分工表.md`
- 各通道自己的知识沉淀文件

## 必须内置

1. `project-startup-collab-suite`
   用于项目启动、读取通讯录、生成启动批次。
2. `agent-init-training-playbook`
   用于首轮培训、职责复述、首个动作对齐。
3. `collab-message-send`
   用于跨通道协作、消息通知和最小回执。
4. `ccr-update-playbook`
   用于维护通讯录、通道真源和协作入口。
5. `skills-governance-upgrade`
   用于技能增删、升级和培训边界判断。
6. `session-health-inspector`
   用于会话健康检查和上下文腐烂判断。
7. `session-rotation-handoff`
   用于会话轮换和接力切换。
8. `task-health-organizer`
   用于任务空间健康检查和知识整理。

## 为什么要单独补这一层

- 通道目录、通讯录和知识沉淀解决“我是谁、我负责什么”
- 公共 skill 解决“我该怎么做”

如果没有这层方法型技能，安装电脑上的 AI 往往只能看到目录，无法稳定完成启动、培训、协作和巡检。

## 当前推荐加载顺序

1. `project-startup-collab-suite`
2. `agent-init-training-playbook`
3. `collab-message-send`
4. 读取自己负责通道目录下的任务和知识沉淀
5. 需要治理时，再加载 `skills-governance-upgrade` / `session-health-inspector` / `task-health-organizer`
