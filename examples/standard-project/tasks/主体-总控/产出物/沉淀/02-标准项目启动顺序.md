# 标准项目启动顺序

## 目标

在用户新电脑上，把 `standard_project` 拉起成一个可继续协作的标准项目。

## 推荐顺序

1. 先读 `examples/standard-project/seed/seed-inventory.json`
2. 再读 `examples/standard-project/seed/ccr_roster_seed.json`
3. 再读 `examples/standard-project/tasks/主体-总控/产出物/沉淀/03-标准项目通讯录与分工表.md`
4. 执行 `python3 scripts/start_standard_project.py`
5. 确认 `standard_project` 已在页面里可见
6. 确认默认启动批次已点亮 6 个核心通道
7. 再决定是否继续点亮扩展治理通道

## 默认启动批次

- `主体-总控`
- `辅助01-结构治理与项目接入`
- `子级01-运行时与后端`
- `子级02-前端与交互`
- `子级03-数据与契约`
- `子级04-测试与验收`

## 第二批可选通道

- `辅助02-文档与知识沉淀`
- `辅助03-用户镜像与业务判断`
- `辅助04-Git桥接与发布同步`
- `辅助05-团队协作Skills治理`
- `辅助06-项目运维`
- `系统01-信息梳理`

## 说明

- 如果当前电脑 `codex` 未就绪，先完成页面和项目初始化，不强行激活 Agent
- 如果出现 CLI 路径错误，优先重跑 `python3 scripts/start_standard_project.py`
- 12 个通道都应在 `ccr_roster_seed.json` 和通讯录/分工表里可查
