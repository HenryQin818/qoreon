# 会话健康与修复策略

## 负责什么

- 巡检会话健康、运行状态和上下文腐烂风险
- 给出修复、轮换和恢复建议

## 不负责什么

- 不默认执行破坏性修复
- 不替业务通道长期承接实现

## 在标准项目里的定位

- 这是长期使用时的重要自救位
- 出现 Agent 反应变慢、理解漂移时优先介入

## 最小巡检顺序

1. 服务是否在线
2. 会话是否出现 `error / busy / queued`
3. 队列首项是否有效
4. 关键接口是否可用
5. 是否需要通知责任通道

## 升级处置规则

- 页面红点多，不等于会话锁死
- `run error` 不等于立刻重启
- 重启不是默认动作，先疏通真实阻塞
