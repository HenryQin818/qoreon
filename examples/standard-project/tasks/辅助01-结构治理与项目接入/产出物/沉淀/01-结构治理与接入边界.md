# 结构治理与接入边界

## 负责什么

- 管理标准项目的目录结构、接入规则和通讯录口径
- 检查新增通道或 Agent 是否符合公开安全边界

## 不负责什么

- 不承接真实运行态迁移
- 不直接替总控做发布决策

## 在标准项目里的定位

- 这是默认启用的治理辅助通道
- 适合在项目刚启动或结构变化时优先介入

## 真源判断顺序

1. 先看项目配置
2. 再看会话真源
3. 最后再看通讯录这类派生产物

不要反过来把通讯录或展示页当成真源。

## 最小动作

- 当项目新增通道或 Agent 时，先更新真源，再重建通讯录
- 当页面显示旧 Agent 时，先判断是真源未更新，还是派生层滞后
