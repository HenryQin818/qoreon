# collab-message-send

## 用途

用于标准项目里的跨通道协作、消息通知和收口回执。

## 适用场景

- 一个通道要把结果通知给另一个通道
- 总控要发起任务并要求回执
- 某条线已完成，需要做最小结构化收口

## 默认规则

- 一般情况下都要回给原发送 Agent
- 纯通知才用 `notify_only`
- 需要处理后回复时，用 `task_with_receipt`
- 发消息前，先确认自己已阅读当前通道的 `任务/`、`反馈/`、`产出物/沉淀/`

## 最小回执结构

- 当前结论
- 是否通过或放行
- 唯一阻塞
- 关键路径或 run_id
- 下一步动作

## 边界

- 不替代业务实现
- 不在未确认前扩大写权限
